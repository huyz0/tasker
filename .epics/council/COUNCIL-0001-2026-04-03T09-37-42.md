## How

The implementation should follow the established **Modular Monolith** structure, updating the Query handlers in each bounded context.

1. **Schema Enhancements**: Ensure proper indexing in MySQL for common filter fields (e.g., status, project_id, owner_id).
2. **TypeSpec Update**: Extend the `main.tsp` contracts to include a standardized `ListRequest` and `ListResponse` with paging metadata.
3. **CQRS Handlers**: Refactor existing `getMany` or `list` query handlers to support Zod-validated sorting and filtering parameters.
4. **Shared Utility**: Create a backend utility for building Drizzle/SQL query clauses from standardized filter objects to prevent duplication.
5. **Frontend/CLI Integration**: Expose these capabilities through the dual-surface CLI (paging flags) and the React GUI.
